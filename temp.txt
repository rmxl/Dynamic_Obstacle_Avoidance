DAgger: Training, Validation & The Best Policy
DAgger Algorithm [Ross et al., 2011]

The Core Intuition

1: Initialize dataset D = ∅, policy π1
2: for iteration n = 1, 2, . . . , N do
3:
Let πmix = βn π ∗ + (1 − βn )πn
4:
Roll out πmix , collect uncertain states Sn
5:
Query Expert for true labels:
6: Dn = {(s, π ∗ (s)) : s ∈ Sn }
7:
Aggregate: D ← D ∪ Dn
8:
Train/Val Split D (e.g., 80/20 split)
9:
Retrain πn+1 on D using early stopping
10:
Test πn+1 in pure simulation (no expert)
11: end for
12: Return best πi based on test performance

When the Learner drives, it inevitably visits uncertain,
off-center states. We pause, ask the MPC how to
recover, and add those exact lessons to D.

Preventing Failure
Mathematical Validation: We split the dataset to
monitor validation loss, preventing the neural
network from memorizing data .
Unassisted Simulation Testing: We evaluate
every iteration in a pure simulation without
expert intervention. Then deploy best iteration.

The Theoretical Guarantee (Ptrain ≈ Ptest )
Because DAgger actively trains the network on its own natural mistakes, the training and testing
distributions align. Result: Errors no longer compound quadratically. The policy error drops to a highly
stable, linear bound of O(ϵT).

Summary

1

Problem: Navigating waypoints among dynamic
obstacles formalized as constrained optimal control.

2

MPC: Receding-horizon optimizer; serves as the
robust expert oracle.

3

IRL: Recovers the latent trajectory reward from
MPC; deploys via RL without a solver.

4

DAgger: Iteratively queries MPC at learner-visited
states to eliminate distribution shift with an O(ϵT )
regret bound.
